# ADR-004: Single Edge Function for Notifications

## Status

Accepted

## Date

2026-06-11

## Context

Precisamos enviar três tipos de notificação:

1. **Daily digest**: Resumo dos jogos do dia (08:00)
2. **Post-match**: Pontos ganhos após jogo finalizado
3. **Deadline reminder**: Aviso 2h antes do kickoff

Opções consideradas:

1. **Função única**: Uma edge function `send-notifications` que recebe o tipo via body
2. **Funções separadas**: Três funções distintas (send-daily-digest, send-post-match, send-deadline-reminder)

## Decision

Usar **uma única edge function** `send-notifications` que recebe o tipo de notificação e os dados via request body.

**Rationale:**
- Menor complexidade de manutenção
- Código compartilhado para envio de push (VAPID, subscriptions)
- Mais fácil de testar e debugar
- Consistente com o padrão do projeto (sync-matches é uma função única)

## Alternatives Considered

### Alternative 1: Funções separadas

- **Description**: Três edge functions independentes
- **Pros**: Isolamento de falhas, deploy independente
- **Cons**: Duplicação de código, mais complexidade
- **Why rejected**: Over-engineering para o escopo atual

## Consequences

### Positive

- Código mais simples e reutilizável
- Menos funções para manter
- Mais fácil de testar

### Negative

- Função maior e mais complexa
- Falha afeta todos os tipos de notificação

### Risks

- **Complexidade crescente**: Função pode ficar muito grande → Mitigação: modularizar em helpers no _shared/
- **Acoplamento**: Mudanças afetam todos os tipos → Mitigação: testes extensivos

## Implementation Notes

```typescript
// POST /functions/v1/send-notifications
// Body: { type: 'daily-digest' | 'post-match' | 'deadline-reminder', data: {...} }

import { sendNotification } from "https://deno.land/x/deno_web_push/mod.ts";

Deno.serve(async (req) => {
  const { type, data } = await req.json();
  
  // Buscar subscriptions do banco
  const { data: subscriptions } = await supabase
    .from('push_subscriptions')
    .select('*');
  
  // Enviar para cada subscription
  for (const sub of subscriptions) {
    await sendNotification({
      subscription: sub,
      message: JSON.stringify({ type, ...data }),
      vapidKeys: { ... },
    });
  }
  
  return new Response(JSON.stringify({ success: true }));
});
```
