# ADR-005: Fixed Brasília Timezone

## Status

Accepted

## Date

2026-06-11

## Context

Notificações agendadas (daily digest às 08:00, deadline reminders) precisam de um timezone consistente. Opções:

1. **Horário de Brasília fixo**: Todos os usuários recebem no mesmo horário absoluto
2. **Timezone por usuário**: Armazenar timezone no perfil e converter

## Decision

Usar **horário de Brasília (America/Sao_Paulo) fixo** para todas as notificações agendadas.

**Rationale:**
- Simplicidade de implementação
- Público-alvo brasileiro (Copa do Mundo)
- Não há usuários internacionais conhecidos
- Evita complexidade de gerenciar timezones por usuário

## Alternatives Considered

### Alternative 1: Timezone por usuário

- **Description**: Armazenar timezone no perfil e converter horários
- **Pros**: Flexibilidade para usuários internacionais
- **Cons**: Complexidade adicional, campo extra no banco
- **Why rejected**: Over-engineering para o público-alvo atual

## Consequences

### Positive

- Implementação mais simples
- Menos campos no banco de dados
- Previsibilidade para usuários

### Negative

- Usuários em outros fusos recebem em horários diferentes
- Não escala para público internacional

### Risks

- **Usuários viajando**: Podem receber notificações em horários estranhos → Mitigação: documentar comportamento

## Implementation Notes

```sql
-- Cron job para daily digest às 08:00 Brasília
SELECT cron.schedule(
  'daily-digest-8am',
  '0 8 * * *',  -- 08:00 UTC-3 (Supabase converte automaticamente)
  $$ SELECT net.http_post(...) $$
);
```

```typescript
// Edge Function: verificar se há jogos hoje (horário de Brasília)
const now = new Date();
const brasiliaTime = new Date(now.toLocaleString("en-US", {timeZone: "America/Sao_Paulo"}));
const today = brasiliaTime.toISOString().split('T')[0];
```
