# ADR-003: injectManifest Service Worker Strategy

## Status

Accepted

## Date

2026-06-11

## Context

O vite-plugin-pwa oferece duas estratégias para service workers:

1. **generateSW**: Gera automaticamente o service worker com Workbox
2. **injectManifest**: Permite criar um service worker customizado e injeta a lista de precache

Atualmente o projeto usa generateSW (padrão), mas para push notifications precisamos de controle total sobre os eventos `push` e `notificationclick`.

## Decision

Migrar para **injectManifest** com um service worker customizado em `src/service-worker.js`.

**Rationale:**
- Controle total sobre eventos push
- Possibilidade de implementar lógica customizada de notificações
- Compatibilidade com Workbox para caching
- Padrão recomendado para PWAs com push notifications

## Alternatives Considered

### Alternative 1: generateSW + extensions

- **Description**: Manter generateSW e adicionar código via workbox.injectManifest ou importScripts
- **Pros**: Menor mudança na configuração atual
- **Cons**: Limitações no controle de eventos push, mais complexo de debugar
- **Why rejected**: Menos flexibilidade para notificações customizadas

## Consequences

### Positive

- Controle total sobre service worker
- Melhor debugabilidade
- Padrão da indústria para PWAs avançados

### Negative

- Requer criar e manter arquivo service-worker.js
- Configuração um pouco mais complexa no vite.config.js

### Risks

- **Erros no service worker**: Pode quebrar caching/offline → Mitigação: testes extensivos
- **Atualizações**: Mudanças no SW requerem cuidado com versionamento → Mitigação: skipWaiting + clientsClaim

## Implementation Notes

```javascript
// vite.config.js
VitePWA({
  strategies: 'injectManifest',
  srcDir: 'src',
  filename: 'service-worker.js',
  // ... resto da configuração
})
```

```javascript
// src/service-worker.js
import { precacheAndRoute } from 'workbox-precaching'
import { clientsClaim } from 'workbox-core'

precacheAndRoute(self.__WB_MANIFEST)
self.skipWaiting()
clientsClaim()

self.addEventListener('push', (event) => {
  const data = event.data.json()
  event.waitUntil(
    self.registration.showNotification(data.title, {
      body: data.body,
      icon: '/pwa-192x192.png',
      data: data.data,
    })
  )
})

self.addEventListener('notificationclick', (event) => {
  event.notification.close()
  event.waitUntil(
    clients.openWindow(event.notification.data.url)
  )
})
```
