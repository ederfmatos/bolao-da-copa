# ADR-002: deno-web-push Library

## Status

Accepted

## Date

2026-06-11

## Context

Precisamos de uma biblioteca para enviar Web Push notifications nas Edge Functions do Supabase (Deno). As opções consideradas foram:

1. **deno-web-push**: Biblioteca Deno nativa para Web Push com VAPID
2. **Implementação manual**: Usar crypto API do Deno + fetch para implementar o protocolo VAPID manualmente
3. **web-push via esm.sh**: Portar a biblioteca npm web-push para Deno via esm.sh

## Decision

Usar **deno-web-push** como biblioteca para enviar Web Push notifications.

**Rationale:**
- Biblioteca Deno nativa, sem dependências npm
- Mantém compatibilidade com o runtime Deno das Edge Functions
- Suporta VAPID authentication nativamente
- API simples e bem documentada

## Alternatives Considered

### Alternative 1: Implementação manual

- **Description**: Implementar VAPID signing e Web Push protocol manualmente usando crypto API do Deno
- **Pros**: Zero dependências externas, controle total
- **Cons**: Complexidade alta, risco de bugs de segurança, tempo de desenvolvimento
- **Why rejected**: Reinventar a roda, risco de erros em criptografia

### Alternative 2: web-push via esm.sh

- **Description**: Usar a biblioteca npm web-push via esm.sh
- **Pros**: Biblioteca madura e bem testada
- **Cons**: Dependência npm em runtime Deno, possível incompatibilidade
- **Why rejected**: Preferência por solução Deno nativa

## Consequences

### Positive

- Integração nativa com Deno
- Menor complexidade de implementação
- Manutenção mais simples

### Negative

- Biblioteca menos madura que web-push (npm)
- Comunidade menor

### Risks

- **Atualizações da biblioteca**: deno-web-push pode não ser atualizado frequentemente → Mitigação: versão fixa no import
- **Compatibilidade**: Pode ter bugs não detectados → Mitigação: testes extensivos

## Implementation Notes

```typescript
import { sendNotification } from "https://deno.land/x/deno_web_push/mod.ts";

await sendNotification({
  subscription: pushSubscription,
  message: JSON.stringify(payload),
  vapidKeys: {
    publicKey: Deno.env.get("VAPID_PUBLIC_KEY"),
    privateKey: Deno.env.get("VAPID_PRIVATE_KEY"),
  },
  subject: "mailto:admin@bolao.com",
});
```
