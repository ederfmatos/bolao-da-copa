# ADR-006: Auto-cleanup Invalid Push Subscriptions

## Status

Accepted

## Date

2026-06-11

## Context

Push subscriptions podem expirar ou se tornar inválidas (ex: usuário desinstalou o app, limpou dados do navegador). Quando tentamos enviar uma notificação para uma subscription inválida, o serviço de push retorna erro 410 Gone.

Opções:

1. **Auto-cleanup**: Remover automaticamente subscriptions que retornam 410
2. **Apenas logar**: Manter todas as subscriptions e apenas logar erros

## Decision

Implementar **auto-cleanup** de subscriptions inválidas.

**Rationale:**
- Mantém o banco de dados limpo
- Evita tentativas repetidas de envio para endpoints inválidos
- Melhora performance (menos queries desnecessárias)
- Padrão recomendado pela Web Push API

## Alternatives Considered

### Alternative 1: Apenas logar

- **Description**: Manter todas as subscriptions e apenas logar erros
- **Pros**: Simplicidade, histórico completo
- **Cons**: Banco inchado, tentativas desnecessárias
- **Why rejected**: Não escala, piora performance

## Consequences

### Positive

- Banco de dados limpo
- Melhor performance de envio
- Menos custos de API (se houver limite)

### Negative

- Perda de histórico de subscriptions
- Se houver bug, pode remover subscriptions válidas

### Risks

- **Falso positivo**: Subscription válida retornando 410 temporariamente → Mitigação: retry antes de remover
- **Bug no código**: Remover subscriptions válidas → Mitigação: testes extensivos, rollback fácil

## Implementation Notes

```typescript
// No send-notifications edge function
const invalidSubscriptions = [];

for (const sub of subscriptions) {
  try {
    await sendNotification({ subscription: sub, ... });
  } catch (error) {
    if (error.statusCode === 410) {
      invalidSubscriptions.push(sub.id);
    }
  }
}

// Remover subscriptions inválidas
if (invalidSubscriptions.length > 0) {
  await supabase
    .from('push_subscriptions')
    .delete()
    .in('id', invalidSubscriptions);
}
```
