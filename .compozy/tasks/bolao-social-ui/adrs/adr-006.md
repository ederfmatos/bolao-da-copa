# ADR-006: Ordenação das Predições na Tela de Detalhes

## Status

Accepted

## Date

2026-06-10

## Context

A tela de detalhes da partida (MatchDetails) exibe uma lista de predições de todos os participantes. É necessário definir como essas predições serão ordenadas para fornecer a melhor experiência do usuário.

## Decision

Ordenar as predições por:
1. **Pontos** (decrescente) — predições com mais pontos aparecem primeiro
2. **Data de criação** (crescente) — predições com a mesma pontuação são ordenadas por quem palpitou primeiro

Esta ordenação se aplica tanto para partidas finalizadas quanto para partidas agendadas/ao vivo. Para partidas não finalizadas, todas as predições têm 0 pontos, então a ordenação será puramente por data de criação.

## Alternatives Considered

### Alternative 1: Ordenar alfabeticamente por nome

- **Description**: Ordenar predições pelo nome do usuário em ordem alfabética
- **Pros**: Fácil de encontrar um usuário específico, previsível
- **Cons**: Não destaca predições bem-sucedidas, menos engajante
- **Why rejected**: Não aproveita o aspecto social de ver quem está indo melhor

### Alternative 2: Ordenar por data de criação (mais recente primeiro)

- **Description**: Mostrar predições mais recentes no topo
- **Pros**: Destaca atividade recente, incentivam palpites rápidos
- **Cons**: Não mostra quem está indo melhor, predições antigas ficam escondidas
- **Why rejected**: Para partidas finalizadas, é mais interessante ver quem acertou primeiro

### Alternative 3: Ordenação diferente para partidas finalizadas vs agendadas

- **Description**: Usar pontos para finalizadas, data para agendadas
- **Pros**: Contextualmente apropriado
- **Cons**: Comportamento inconsistente, mais complexo de implementar
- **Why rejected**: A ordenação por pontos + data funciona bem para ambos os casos

## Consequences

### Positive

- Para partidas finalizadas, usuários veem imediatamente quem acertou melhor
- Para partidas agendadas, usuários veem quem palpitou primeiro (pode incentivar palpites rápidos)
- Ordenação consistente em todos os tipos de partida
- Simples de implementar (ORDER BY points DESC, created_at ASC)

### Negative

- Para partidas agendadas, a ordem cronológica pode não ser tão interessante
- Usuários que palpitaram por último aparecem no final da lista

### Risks

- **Risk**: Usuários podem não entender por que a ordem é aquela
  - **Mitigation**: Adicionar indicador visual claro de pontos na interface

## Implementation Notes

A ordenação será implementada na view `match_predictions` ou na query do hook `useMatchPredictions`:

```sql
ORDER BY points DESC, created_at ASC
```
