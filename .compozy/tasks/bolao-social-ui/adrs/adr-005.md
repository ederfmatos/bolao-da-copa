# ADR-005: Redesign completo em uma fase com Tailwind CSS

## Status

Accepted

## Date

2026-06-10

## Context

O projeto Bolão Copa 2026 possui uma UI funcional mas visualmente básica, com:
- 100% inline styles sem CSS framework
- Apenas features single-user (sem ver palpites dos outros)
- Sem navegação fixa (bottom nav)
- Sem suporte a dark/light mode

O usuário deseja:
1. Ver palpites de outros participantes (sempre visível)
2. Design mais moderno com dark/light mode + toggle
3. Tailwind CSS como framework de estilização
4. Bottom navigation bar
5. Palpites dos outros na tela de detalhes da partida

## Decision

Realizar redesign completo em uma única fase:
- Migrar todos os inline styles para Tailwind CSS
- Implementar dark/light mode com toggle global
- Adicionar bottom navigation bar fixa
- Criar tela de detalhes da partida com lista de palpites dos outros
- Atualizar RLS para permitir leitura de todos os palpites
- Palpites sempre visíveis (sem restrição de tempo)

## Alternatives Considered

### Alternative 1: Redesign incremental por página

- **Description**: Migrar uma página por vez para Tailwind, dark mode disponível apenas nas páginas já migradas
- **Pros**: Menos risco por entrega, rollback fácil por página
- **Cons**: Experiência inconsistente durante migração, dark mode parcial confunde usuários
- **Why rejected**: Usuário prefere resultado coeso e consistente

### Alternative 2: Social-first, visual depois

- **Description**: Priorizar features sociais (ver palpites dos outros) primeiro, redesign visual como segunda fase
- **Pros**: Features sociais disponíveis mais rápido
- **Cons**: UX visual fica para depois, possível retrabalho ao integrar design final
- **Why rejected**: Usuário quer ambas as melhorias juntas

## Consequences

### Positive

- Resultado visual coeso e consistente em todo o app
- Dark/light mode disponível desde o início
- Experiência mobile-first com bottom navigation
- Features sociais (ver palpites) integradas ao novo design
- Código mais manutenível com Tailwind vs inline styles

### Negative

- Maior esforço inicial antes de qualquer entrega visual
- Risco de scope creep ao refazer tudo de uma vez
- Possível regressão visual temporária durante migração

### Risks

- **Risk**: Migração demorar mais que o esperado
  - **Mitigation**: Priorizar páginas críticas (Matches, Predict, Leaderboard) primeiro
- **Risk**: Inconsistências visuais entre páginas durante desenvolvimento
  - **Mitigation**: Criar design tokens/componentes base antes de migrar páginas

## Implementation Notes

1. Instalar Tailwind CSS e configurar dark mode
2. Criar componentes base (Button, Card, Input) com Tailwind
3. Implementar ThemeProvider para dark/light mode
4. Atualizar RLS: criar policy `predictions_read_all` para usuários autenticados
5. Criar hook `useMatchPredictions(matchId)` para buscar palpites de todos
6. Criar tela `MatchDetails.jsx` com lista de palpites
7. Adicionar bottom navigation ao App.jsx
8. Migrar páginas uma a uma para Tailwind
