# ADR-002: Supabase como plataforma principal

## Status
Accepted

## Context
Precisamos de um backend que forneça: banco de dados PostgreSQL, autenticação, funções serverless, e real-time subscriptions. Avaliamos opções como Firebase, AWS Amplify, e Supabase.

## Decision
Usar Supabase como plataforma única para:
- **PostgreSQL**: banco de dados relacional com RLS nativo
- **Auth**: Google OAuth integrado, sem gerenciamento de senhas
- **Edge Functions**: Deno runtime para sync de dados de futebol
- **Realtime**: subscriptions para atualizações do leaderboard
- **pg_cron**: agendamento de tarefas (sync a cada 5 minutos)

## Consequences
- **Positivo**:
  - Stack unificado reduz complexidade operacional
  - RLS garante segurança no nível do banco
  - Realtime nativo sem necessidade de WebSockets customizados
  - Free tier generoso para MVP
- **Negativo**:
  - Vendor lock-in com Supabase
  - Edge Functions em Deno (menos familiar que Node.js)
- **Risco**: Limites do free tier se o número de participantes crescer muito
