# ADR-003: Multi-provider com fallback para dados de futebol

## Status
Accepted

## Context
Precisamos de uma fonte confiável de dados das partidas da Copa do Mundo 2026 (placares, horários, status). APIs de futebol podem ter downtime, rate limits, ou custos variáveis.

## Decision
Implementar uma arquitetura multi-provider com fallback em cadeia:
1. **Primary**: football-data.org (API gratuita, cobre Copa do Mundo)
2. **Fallback**: API-Football via RapidAPI (paga, mas confiável)

A função `fetchWithFallback` tenta cada provider em ordem e retorna o primeiro resultado bem-sucedido. Se todos falharem, lança um erro.

Interface `FootballProvider` padroniza o contrato, permitindo adicionar novos providers facilmente.

## Consequences
- **Positivo**:
  - Alta disponibilidade: se um provider falhar, o outro assume
  - Extensível: adicionar um terceiro provider requer apenas implementar a interface
  - Isola lógica de mapeamento de cada API
- **Negativo**:
  - Complexidade adicional na camada de integração
  - Necessidade de manter chaves de API para múltiplos serviços
- **Risco**: Ambos providers podem falhar simultaneamente (mitigado por logs e alertas)
