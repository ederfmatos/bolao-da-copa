# ADR-001: Página de Perfil Dedicada para Visualização de Palpites

## Status

Accepted

## Date

2026-06-10

## Context

Atualmente, os usuários podem ver os palpites de outros participantes apenas no contexto de uma partida específica (tela de detalhes da partida) ou no ranking geral (leaderboard). Não existe uma forma de ver o histórico completo de palpites de um usuário específico, o que limita a análise de desempenho e a comparação entre participantes.

A infraestrutura atual já suporta essa funcionalidade:
- RLS permite leitura pública de predictions e profiles
- Componentes Avatar e PredictionRow já existem e são reutilizáveis
- A view `match_predictions` já junta predictions com profiles

## Decision

Criar uma página de perfil dedicada acessível via `/user/:userId` com:
- Cabeçalho com avatar, nome, pontos totais e posição no ranking
- Seção de estatísticas resumidas (total de palpites, distribuição por pontuação, taxa de placar exato)
- Lista cronológica de todos os palpites do usuário (do mais recente para o mais antigo)

Pontos de acesso:
- Leaderboard: clicar no nome/avatar de um usuário abre o perfil
- Lista de palpites da partida: clicar no nome/avatar de um usuário abre o perfil

## Alternatives Considered

### Alternative 1: Modal/Drawer lateral

- **Description**: Ao clicar em um nome, abre modal ou drawer lateral sobre a tela atual com as informações do perfil
- **Pros**: Menos interrupção, mantém contexto da tela anterior, implementação mais rápida
- **Cons**: Espaço limitado em mobile, difícil de mostrar histórico longo, não permite compartilhamento de URL
- **Why rejected**: Experiência limitada em mobile, não permite deep linking ou compartilhamento de perfil

### Alternative 2: Expandir inline na lista de palpites

- **Description**: Ao clicar em um nome na lista de palpites, expande seção abaixo com os palpites daquele usuário
- **Pros**: Sem navegação nova, tudo na mesma tela, implementação simples
- **Cons**: Limitado em espaço, não funciona pelo leaderboard, difícil de mostrar estatísticas
- **Why rejected**: Muito limitado, não atende o requisito de acesso pelo leaderboard, experiência pobre em mobile

## Consequences

### Positive

- Experiência completa e dedicada para visualização de perfil
- Permite deep linking e compartilhamento de perfil via URL
- Escalável para futuras features (comparação entre usuários, histórico por fase, etc.)
- Consistente com padrões de apps modernos (perfis dedicados)
- Reutiliza componentes existentes (Avatar, PredictionRow)

### Negative

- Mais trabalho de implementação comparado a modal/inline
- Nova rota e página para manter
- Necessidade de criar novo hook para buscar predições do usuário

### Risks

- **Risk**: Performance ao carregar muitos palpites (72+ partidas)
  - **Mitigation**: Implementar paginação ou virtualização se necessário
- **Risk**: Navegação confusa se não houver botão "Voltar" claro
  - **Mitigation**: Usar `navigate(-1)` para preservar contexto e filtros

## Implementation Notes

- Criar hook `useUserPredictions(userId)` que busca predições do usuário juntando com matches
- Criar componente `UserProfile.jsx` com cabeçalho, estatísticas e lista
- Adicionar rota `/user/:userId` em App.jsx
- Tornar nomes/avatars clicáveis no Leaderboard e PredictionRow
- Usar `navigate(-1)` no botão "Voltar" para preservar contexto
